# 年报页面布局优化设计说明

## 1. 背景

当前年报已具备两套阅读方式：网格卡片用于快速扫数据，海报流用于沉浸式年度总结与导出。但它们落在现有报告页后，顶部控制区承担了过多职责：报告类型切换、年份选择、导出风格、预览、刷新、重建、导出同时出现，导致页面在窄窗口或低高度窗口下显得拥挤，并压缩年报主体阅读空间。

本优化采用“顶部瘦身 + 年报内工具条”的结构，保留现有功能能力，同时让页面层级更贴近播放器的工具型主界面。

## 2. 目标

- 报告页顶部只承载跨月报/年报共用的主路径控制。
- 年报专属的视图、导出风格、预览、重建缓存下沉到年报内容区。
- 年报默认优先呈现可扫描的数据报告，海报流作为沉浸展示与分享导出入口。
- 在桌面、窄窗口、低高度窗口下减少控制区对内容区的挤压。

## 3. 信息架构调整

### 3.1 外层报告页顶部

保留：

- `月报 / 年报` 切换
- 年报模式下的年份选择
- `刷新`
- `导出 PNG`

移出：

- 年报导出风格选择
- 年报导出预览
- 年报重建缓存

外层顶部的职责是选择“看哪份报告”和执行最常用动作，不再承载年报专属细节。

### 3.2 年报内容区工具条

在年报内容区顶部提供轻量工具条：

- 左侧：`数据报告 / 海报流` 视图切换
- 右侧：导出风格选择、预览、重建缓存

该工具条只在年报有数据时展示。它的职责是选择“年报怎么看”和管理年报专属动作。

### 3.3 年报默认视图

年报默认进入 `数据报告`，即当前网格卡片视图。海报流保留为可切换的沉浸模式。

原因：

- 数据报告更符合当前播放器界面的工具型阅读习惯。
- 用户进入报告页时能快速看到核心指标和排行。
- 海报流保留年度总结的仪式感，适合分享、预览、导出前查看。

## 4. 交互细节

- 切换 `月报 / 年报` 时仍保持同一路由和同一报告容器。
- 切换年份后刷新年报数据，保持当前年报视图模式不变。
- `导出 PNG` 使用当前选择的年报导出风格；若用户在年报工具条中切换风格，外层导出按钮立即遵循新风格。
- `预览` 打开现有导出预览弹窗。
- `重建缓存` 在年报工具条中触发，按钮状态沿用当前 loading / disabled 逻辑。
- 年报视图切换不重取数据，只切换展示组件。

## 5. 视觉与响应式原则

- 外层顶部主控保持一行优先，允许换行但不应成为高密度按钮堆。
- 年报工具条采用轻量容器，不做强视觉卡片，避免形成“卡片套卡片”的观感。
- 窄窗口下年报工具条可上下分行：视图切换在前，导出与维护动作在后。
- 低高度窗口下海报流继续优先压缩或隐藏次要导航，确保当前页主体可读。
- 海报流的章节、阶段导航保留，但不作为默认阅读路径的负担。

## 6. 实施边界

本轮仅调整前端布局与交互编排，不改变年报数据模型、聚合逻辑、IPC 合同、导出生成能力。

涉及文件预计为：

- `src/renderer/views/Report/index.vue`
- `src/renderer/views/ReportYearly/index.vue`
- `src/renderer/views/ReportYearly/components/PosterStory.vue`
- 必要时微调 `src/renderer/store/reportYearly/state.ts`

## 7. 验收标准

- 报告页顶部在年报模式下不再出现导出风格、预览、重建缓存。
- 年报内容区提供 `数据报告 / 海报流` 切换，默认显示数据报告。
- 年报专属导出风格、预览、重建缓存从年报内容区操作。
- 外层 `导出 PNG` 仍可正常导出当前年报风格。
- 窄窗口与低高度窗口下控制区不明显压缩主体内容，海报流内容仍可滚动阅读。

# 年报 Poster 导出图自适应布局设计说明

## 1. 背景

当前年报 `poster` 导出图已经完成一轮密度优化，整体比初版更紧凑。但它仍然主要依赖固定区块高度与固定条目数：

- Hero 区高度固定
- 中段 `Year Favorites + KPI` 高度固定
- `Replay Songs` 固定展示 5 条
- Footer 高度固定

这让导出图在“标准数据量”下表现不错，但遇到数据量明显偏少、偏多，或文本长度差异较大时，仍然容易出现两种问题：

- 数据少时，某些区块显得偏空
- 数据多或文本长时，局部区域会开始显得紧

本次优化不再继续单纯手调固定坐标，而是引入“区块高度自适应”规则：保持海报式构图和固定画布尺寸不变，但允许部分区块在明确边界内按数据量自动伸缩。

## 2. 目标

- 让 `poster` 导出图在不同年份、不同数据量下都尽量不空、不挤
- 保持 `1080 x 1920` 导出尺寸与当前导出链路不变
- 保持现有海报结构，不通过隐藏核心区块来回避留白
- 将自适应控制在可预测的规则范围内，避免导出图在不同年份之间风格漂移过大

## 3. 设计原则

### 3.1 稳定优先

导出图首先是一张“年度海报”，不是自由流式页面。  
因此自适应只发生在有限的几个布局参数上，不改变整体信息结构和阅读顺序。

### 3.2 伸缩有边界

每个允许伸缩的区块都必须有：

- 最小高度
- 标准高度
- 最大高度

避免某个年份的数据极少时区块塌得太薄，或数据偏多时区块无限膨胀。

### 3.3 Replay 区是主伸缩区

本次主要把自适应能力放在 `Replay Songs` 区，因为它最直接受到条目数量影响，也是当前最容易制造大面积留白的地方。

### 3.4 Hero 区保持稳定

Hero 区承担整张图的第一视觉锚点，结构和高度变化应尽量小，只允许轻微调整，不做大幅波动。

## 4. 自适应结构

导出图仍然维持四段结构：

1. Hero
2. Middle Section
3. Replay Section
4. Footer

但各段的适应性不同。

### 4.1 Hero 区

Hero 区继续承载：

- 年报标题
- 年份副标题
- 总收听时长
- 总收听次数

规则：

- 高度保持相对稳定
- 只允许小范围浮动，用于微调主数字与副文案的呼吸关系
- 不因数据量多少而大幅缩放

建议实现为“固定主高度 + 小范围补偿”，而不是完全动态。

### 4.2 Middle Section

Middle Section 继续采用：

- 左栏：年度歌曲 / 年度歌手 / 年度专辑
- 右栏：活跃天数 / 新歌占比 / 新歌手占比

规则：

- 默认保持当前双栏结构
- 高度允许在一个中等范围内浮动
- 主要根据以下因素决定是否增减高度：
  - 年度专辑是否缺失
  - 左栏文本是否明显偏长
  - 右栏 KPI 是否需要更紧或更松的垂直节奏

不建议通过“缺少专辑就隐藏整行”的方式处理，而是让整体区块轻度收缩，保持结构完整。

### 4.3 Replay Section

Replay Section 是本次主要自适应区。

规则：

- 可见条目数根据实际数据量决定，但保持上限
- 区块高度根据可见条目数和单行高度共同计算
- 单行高度也允许在有限范围内浮动

建议行为：

- `1-2` 条时：区块明显收短
- `3-5` 条时：使用标准布局
- `6+` 条时：最多展示到预设上限，并通过更紧凑行高容纳

这样导出图在条目偏少时不会留下明显空洞，在条目偏多时也不会把整体节奏打乱。

### 4.4 Footer

Footer 继续只承担：

- 一句收尾文案
- 生成时间
- 品牌标识

规则：

- Footer 高度不再完全固定
- 在一个最小值和最大值之间浮动
- 优先吸收剩余空间，但不能膨胀成新的主视觉区块

Footer 的适应目标不是“承载更多信息”，而是平滑收口整张海报。

## 5. 自适应参数建议

本次不把具体数字写死在设计层，但实现上应显式计算以下参数：

- `heroHeight`
- `middleHeight`
- `replayVisibleCount`
- `replayRowHeight`
- `replayHeight`
- `footerHeight`

这些参数应先统一计算，再进入实际绘制流程。  
不建议把规则散落在各个 `fillText` / `fillGlassCard` 调用之间。

## 6. 文本适配原则

为了配合高度自适应，文本处理规则也需要同步明确：

- 歌名、歌手名、专辑名继续使用截断策略
- 截断长度可根据区块密度轻微调整
- 优先保证单行稳定，不优先追求显示完整文本
- 当数据较少导致区块收缩时，不额外放大文本来填空

也就是说，文本负责“稳定排版”，高度负责“吸收差异”，两者不要互相抢职责。

## 7. 实施边界

本次仅调整：

- `src/renderer/store/reportYearly/action.ts` 中 `drawYearlyPoster()` 的布局计算规则
- 与之相关的条目可见数、区块高度、单行高度、截断长度

本次不调整：

- `drawYearlyPosterClassic()`
- 年报页面内浏览布局
- 数据 DTO、聚合逻辑、IPC、缓存结构
- 导出尺寸与导出入口
- 通过隐藏核心区块改变导出信息结构

## 8. 验收标准

- 数据较少时，导出图不会出现明显的大块留白
- 数据中等时，导出图保持当前成熟版式的稳定感
- 数据较多时，`Replay Songs` 区不会挤坏整体布局
- Hero 仍然是第一视觉焦点
- Middle Section 仍然保持完整双栏结构
- Footer 只负责收尾，不会膨胀成新的内容区
- 同一套规则应用到不同年份时，导出图整体风格保持一致
